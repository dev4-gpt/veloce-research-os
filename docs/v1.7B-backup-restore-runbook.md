# V1.7B Backup and Restore Runbook

Date: 2026-05-20

## Purpose

This runbook creates a reproducible, non-destructive backup and restore process for the Veloce VPS stack.

The goal is production hardening, not a destructive restore drill. Restore commands below write into a test directory unless an operator explicitly approves a production restore.

## Scope

Included:

```text
/root/veloce-research-os
/root/ai-agency compose and env metadata inventory
/opt/veloce-ruflo-sandbox hardened Ruflo sandbox config
Docker volume inventory
Paperclip container-visible artifact inventory
Open WebUI persistent volume
Hermes-related config and health evidence
MCPO proxy config and verification evidence
```

Not included by default:

```text
raw secrets printed into logs
live production restore over existing volumes
provider-side hosted data
browser profiles or local Mac Obsidian vault
GitHub remote backup beyond git remotes
```

## RPO and RTO

Targets for the operator beta:

```text
RPO: <= 24 hours
RTO: <= 60 minutes for service config and repo recovery
```

These are targets, not guarantees, until one restore drill is completed.

## Backup Directory

```bash
export BACKUP_TS="$(date -u +%Y%m%dT%H%M%SZ)"
export BACKUP_ROOT="/root/veloce-backups/${BACKUP_TS}"
mkdir -p "$BACKUP_ROOT"
```

## Inventory Commands

Run on the VPS:

```bash
cd /root

docker ps --format 'table {{.Names}}\t{{.Image}}\t{{.Status}}\t{{.Ports}}' \
  | tee "$BACKUP_ROOT/docker-ps.txt"

docker volume ls \
  | tee "$BACKUP_ROOT/docker-volumes.txt"

docker network ls \
  | tee "$BACKUP_ROOT/docker-networks.txt"

docker inspect aiagency-openwebui aiagency-mcpo-openwebui-proxy aiagency-mcpo-time aiagency-hermes \
  > "$BACKUP_ROOT/aiagency-container-inspect.json" 2>/dev/null || true

docker inspect paperclip-iraj-paperclip-1 \
  > "$BACKUP_ROOT/paperclip-container-inspect.json" 2>/dev/null || true
```

## Repository and Config Backup

Do not print `.env` contents to terminal. Archive them directly.

```bash
tar -czf "$BACKUP_ROOT/veloce-research-os.tar.gz" \
  -C /root veloce-research-os

tar \
  --exclude='ai-agency/.git' \
  --exclude='ai-agency/**/__pycache__' \
  -czf "$BACKUP_ROOT/ai-agency-config.tar.gz" \
  -C /root ai-agency

if [ -d /opt/veloce-ruflo-sandbox ]; then
  tar -czf "$BACKUP_ROOT/veloce-ruflo-sandbox.tar.gz" \
    -C /opt veloce-ruflo-sandbox
fi
```

## Docker Volume Backup

List candidate volumes:

```bash
docker volume ls --format '{{.Name}}' | sort
```

Create archives:

```bash
for volume in $(docker volume ls --format '{{.Name}}' | sort); do
  safe_name="$(echo "$volume" | tr '/:' '__')"
  docker run --rm \
    -v "${volume}:/volume:ro" \
    -v "$BACKUP_ROOT:/backup" \
    alpine:3.20 \
    sh -lc "cd /volume && tar -czf /backup/docker-volume-${safe_name}.tar.gz ."
done
```

## Checksums

```bash
cd "$BACKUP_ROOT"
find . -maxdepth 2 -type f ! -name SHA256SUMS -print0 | sort -z | xargs -0 sha256sum > SHA256SUMS
sha256sum -c SHA256SUMS
```

## Non-Destructive Restore Test

Create a restore test directory:

```bash
export RESTORE_TS="$(date -u +%Y%m%dT%H%M%SZ)"
export RESTORE_ROOT="/root/veloce-restore-test/${RESTORE_TS}"
mkdir -p "$RESTORE_ROOT"
```

Extract repo/config archives into the test path:

```bash
tar -xzf "$BACKUP_ROOT/veloce-research-os.tar.gz" -C "$RESTORE_ROOT"
tar -xzf "$BACKUP_ROOT/ai-agency-config.tar.gz" -C "$RESTORE_ROOT"

if [ -f "$BACKUP_ROOT/veloce-ruflo-sandbox.tar.gz" ]; then
  tar -xzf "$BACKUP_ROOT/veloce-ruflo-sandbox.tar.gz" -C "$RESTORE_ROOT"
fi
```

Inspect without printing secrets:

```bash
find "$RESTORE_ROOT" -maxdepth 3 -type f | sort | sed -n '1,200p'
test -f "$RESTORE_ROOT/veloce-research-os/README.md"
test -f "$RESTORE_ROOT/ai-agency/docker-compose.yml"
test -f "$RESTORE_ROOT/veloce-ruflo-sandbox/.claude-flow/config.yaml" || true
```

Test a single Docker volume archive into a local directory, not a live volume:

```bash
mkdir -p "$RESTORE_ROOT/volume-sample"
first_volume_archive="$(find "$BACKUP_ROOT" -maxdepth 1 -name 'docker-volume-*.tar.gz' | sort | head -n 1)"
if [ -n "$first_volume_archive" ]; then
  tar -xzf "$first_volume_archive" -C "$RESTORE_ROOT/volume-sample"
  find "$RESTORE_ROOT/volume-sample" -maxdepth 2 -type f | sort | sed -n '1,80p'
fi
```

## Production Restore Guardrail

Do not restore over production volumes unless:

```text
1. The operator explicitly approves the restore.
2. Current production volumes are snapshotted first.
3. The target service is stopped.
4. The restore archive checksum passes.
5. A rollback archive exists.
6. Post-restore health checks are ready to run.
```

## Rollback

If a backup command fails:

```text
Do not delete existing production data.
Keep the failed backup directory for inspection.
Start a new timestamped backup directory for the retry.
```

If a non-destructive restore test fails:

```text
Remove only /root/veloce-restore-test/<timestamp>.
Do not touch production volumes.
Re-run checksum verification.
Record the failed archive name and command output in Paperclip.
```
