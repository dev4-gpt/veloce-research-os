# V1.8 Traefik Route Repair

Date: 2026-05-20

## Context

After a full `docker compose up -d --build --force-recreate`, public routes temporarily failed:

```text
https://chat.srv1314350.hstgr.cloud -> 404
https://tools.srv1314350.hstgr.cloud/openapi.json -> 404
https://hermes.srv1314350.hstgr.cloud/health -> Bad Gateway
```

Internal service checks were healthy after repair.

## Root Cause

The recreated application containers needed explicit Traefik network selection.

Traefik runs in host network mode on this Hostinger stack, so it cannot be connected to `aiagency`. The correct fix is to keep app containers on `aiagency` and set:

```yaml
- traefik.docker.network=aiagency
```

on public Traefik-routed app containers.

## Repair Applied

On the VPS, `/root/ai-agency/docker-compose.yml` was backed up and patched so core public services include:

```yaml
- traefik.docker.network=aiagency
```

Traefik was restarted after service recreation:

```bash
docker restart traefik-traefik-1
```

## Rule

Do not try to attach `traefik-traefik-1` to `aiagency`; it uses host networking and Docker will reject that.

Patch routed application containers instead, then restart Traefik.
