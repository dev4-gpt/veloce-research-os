# V1.9M Operating Graph Ranking Boost

Date: 2026-05-20

## Context

V1.9L added curated operating-memory files under `knowledge/graph-memory/`. After VPS regeneration, Graphify ingested those files and the graph grew to 378 nodes and 523 edges.

The first live query proved the memory existed, but top matches still favored older docs such as `docs/setup-log.md` while V1.9L and operating-memory files appeared mostly in relationships. V1.9M fixes the ranking layer so generated operating memory becomes first-class evidence.

## What Changed

`/knowledge_graph_query` now treats `knowledge/graph-memory/` as a first-class source kind:

```text
source_filter = docs      -> includes docs/ and knowledge/graph-memory/
source_filter = knowledge -> only knowledge/graph-memory/
source_filter = code      -> code files
source_filter = tests     -> test files
source_filter = all       -> every graph source
```

Ranking now boosts:

- `knowledge/graph-memory/*`
- `knowledge/graph-memory/veloce-operating-graph.md`
- `docs/v1.9L-operating-graph-ingestion.md`
- queries containing `operating graph`
- product terms such as Paperclip, Hermes, Ruflo, OpenWebUI, MCPO, Obsidian, and Graphify

## Recommended Query

```bash
curl -sS https://tools.srv1314350.hstgr.cloud/knowledge_graph_query \
  -H "Authorization: Bearer $MCPO_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"question":"what is the Veloce operating graph across Paperclip, Hermes, Ruflo, OpenWebUI, MCPO, Obsidian, and Graphify?","source_filter":"docs","answer_style":"summary","max_results":8}' \
  | python3 -m json.tool
```

Expected top evidence should now include `knowledge/graph-memory/veloce-operating-graph.md` or related graph-memory files.

## Why This Matters

Users should not have to know repo internals to analyze Veloce. The graph should answer from product memory first:

```text
OpenWebUI tool surface
MCPO endpoint surface
Paperclip work ledger
Hermes/Ruflo skill layer
Obsidian/Graphify memory loop
VPS/GitHub verification context
```

That makes the graph feel like an operating memory layer, not merely a code index.
