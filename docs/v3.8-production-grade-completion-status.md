- `/api/health` returns JSON health.
- `/api/issues/VEL-v2.0F-PILOT` returns JSON `404 Issue not found`.

That means the next safe production step is not to run live writeback yet. First create or confirm the exact pilot issue `VEL-v2.0F-PILOT`, then provision a native scoped automation token. Browser session cookies and direct database writes remain disallowed as the production automation path.

If Paperclip auto-generates a numeric issue id such as `VEL-145`, use it through `PAPERCLIP_PILOT_ISSUE_ID`. V3.2 discovery and writeback will lock to that exact generated id for the run and reject invalid or mismatched IDs.
