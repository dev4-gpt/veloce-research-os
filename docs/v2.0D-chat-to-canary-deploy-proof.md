# V2.0D Chat-to-Canary Deploy Proof

V2.0D proves the canary-deploy control shape without mutating production.

The goal is intentionally narrow:

```text
approved canary intent -> no-op canary candidate -> pre-health snapshot -> post-health snapshot -> rollback packet -> alert packet -> audit ledger -> graph memory packet
```

This is not a live service deployment. It is the control-plane proof that must exist before a real canary can be allowed.

## Default Dry Run

```bash
cd projects/reliability-policy-matrix
make canary-deploy-proof
```

The default config is:

```text
configs/canary_deploy_v2_0D.json
```

By default, `live_enabled` is `false`. The proof writes only local derived artifacts.

## Live Gates

Live-ready mode is blocked unless all gates pass:

```text
VELOCE_CANARY_DEPLOY_LIVE=1
live_enabled=true
VELOCE_CANARY_APPROVED is set
```

Even in live-ready mode, the V2.0D proof does not mutate production. It only writes the canary packet, rollback packet, alert packet, audit ledger, and graph-memory packet.

## Scoped Actions

The proof writes:

```text
one no-op canary candidate packet
one pre-health snapshot
one post-health snapshot
one rollback packet
one alert packet
one JSONL audit record
one Obsidian/Graphify memory packet
```

The committed candidate is a no-op compose candidate. Real Docker, Traefik, VPS, and production writes remain outside V2.0D.

## Outputs

```text
artifacts/derived/canary_deploy_v2_0D.json
artifacts/derived/canary_deploy_v2_0D.md
artifacts/derived/canary_deploy_audit_v2_0D.jsonl
artifacts/derived/canary_deploy_memory_v2_0D.md
artifacts/derived/canary_deploy_packet_v2_0D.json
artifacts/derived/canary_rollback_packet_v2_0D.json
artifacts/derived/canary_alert_packet_v2_0D.json
```

The memory markdown is intended for the Obsidian -> Graphify loop, so the proof becomes queryable through OpenWebUI graph memory after extraction.

## Rollback

If the canary candidate should not proceed, the rollback is:

```text
do not promote canary
use the rollback packet to restore last-known-good
verify stack health
record correction in graph memory
```

After V2.0D is accepted, the next proof is autonomous rollback execution, still dry-run and packet-first.
