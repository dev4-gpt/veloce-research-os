# V1.9K Graph Query Quality Upgrade

Date: 2026-05-20

## Context

V1.9J exposed the Graphify knowledge graph through OpenWebUI and MCPO. The first implementation proved the connection, but broad product questions could return low-level test or code nodes before the architecture/runbook documents a user actually needed.

V1.9K upgrades the query layer while keeping Graphify unchanged. The graph remains the machine layer. Obsidian remains the human-readable memory layer. MCPO now ranks and summarizes graph results in a way that is useful from chat.

## What Changed

`/knowledge_graph_query` now accepts:

```json
{
  "question": "what connects OpenWebUI, Hermes, Ruflo, Paperclip, MCPO, Obsidian, and Graphify?",
  "max_results": 8,
  "source_filter": "docs",
  "answer_style": "summary",
  "include_relationships": true
}
```

New behavior:

- Docs-first ranking is the default for product, architecture, memory, and showcase questions.
- `source_filter` can target `docs`, `knowledge`, `code`, `tests`, or `all`.
- `max_results` is clamped to 1-20.
- Test files are penalized unless the query asks for tests/proof/validation or `source_filter="tests"`.
- Product terms such as OpenWebUI, Hermes, Ruflo, Paperclip, MCPO, Obsidian, and Graphify receive ranking weight.
- Responses keep legacy fields and add `source_filter`, `ranking_mode`, and `summary_answer`.

## Human And Agent Layers

The graph UI exists for humans: Obsidian can display the vault and graph relationships visually.

The graph query endpoint exists for agents and chat: OpenWebUI can ask graph-grounded questions without reading the full repo or hallucinating from general model knowledge.

The intended product path is:

```text
OpenWebUI -> Veloce Agentic Control -> MCPO -> Graphify graph.json
Obsidian -> human-readable memory
Hermes/Ruflo -> consume returned graph context for reasoning and planning
```

## Recommended OpenWebUI Prompt

```text
Use the Veloce Agentic Control tool to run veloce_knowledge_graph_query.
Arguments:
question: what connects OpenWebUI, Hermes, Ruflo, Paperclip, MCPO, Obsidian, and Graphify?
max_results: 8
source_filter: docs
answer_style: summary
include_relationships: true
Return only the raw tool output.
```

## Acceptance

- Product/architecture graph questions return docs before tests.
- `source_filter="docs"` excludes code and test matches from `matches`.
- `source_filter="tests"` returns test evidence when requested.
- Responses preserve `ok`, `service`, `trace_id`, `source`, `matches`, `relationships`, `evidence_docs`, and `next_action`.
- Responses add `source_filter`, `ranking_mode`, and `summary_answer`.
- Raw graph files remain private until a separate public-safety scan and redaction pass.
