# V1.7C Monitoring and Alerting Runbook

Date: 2026-05-20

## Purpose

This runbook implements the first production-readiness health monitor for the Veloce stack.

It is a read-only monitor. It does not mutate Paperclip, GitHub, Docker, or Ruflo runtime state.

## Files

```text
projects/reliability-policy-matrix/scripts/stack_health_monitor.py
projects/reliability-policy-matrix/configs/stack_health_v1_7C.json
projects/reliability-policy-matrix/Makefile
projects/reliability-policy-matrix/README.md
```

## Checks

The monitor verifies:

```text
chat.srv1314350.hstgr.cloud returns HTTP 200
tools.srv1314350.hstgr.cloud/openapi.json returns HTTP 200 and OpenAPI 3.0.3
MCPO /stack_status returns ok=true
MCPO /repo_status returns ok=true
MCPO /ruflo_status returns ok=true and production_enabled=false
MCPO /ruflo_execution_packet returns approval_gated_execution_packet_ready
```

## Run

Run on the VPS:

```bash
cd /root/veloce-research-os
git pull --ff-only

cd /root/ai-agency
set -a
source .env
set +a

cd /root/veloce-research-os/projects/reliability-policy-matrix
make stack-health
```

Direct Python path:

```bash
python3 scripts/stack_health_monitor.py --config configs/stack_health_v1_7C.json
```

Expected report:

```text
artifacts/derived/stack_health_v1_7C.json
```

Expected top-level output:

```json
{
  "ok": true
}
```

## Proxy Log Proof

Run after the monitor:

```bash
docker logs --tail=100 aiagency-mcpo-openwebui-proxy \
  | grep -E 'stack_status|repo_status|ruflo_status|ruflo_execution_packet'
```

Expected:

```text
POST /stack_status HTTP/1.1" 200
POST /repo_status HTTP/1.1" 200
POST /ruflo_status HTTP/1.1" 200
POST /ruflo_execution_packet HTTP/1.1" 200
```

## Failure Interpretation

```text
openwebui_public fails:
  Check Traefik route, Open WebUI container health, and chat domain DNS.

mcpo_openapi fails:
  Check Traefik tools route and mcpo-openwebui-proxy container.

mcpo_stack_status fails:
  Treat as stack health regression. Inspect response JSON before restarting services.

mcpo_repo_status fails:
  Check /workspace/veloce-research-os mount in the proxy and git availability.

mcpo_ruflo_status fails:
  Check /opt/veloce-ruflo-sandbox mount and hardening files.

mcpo_ruflo_execution_packet fails:
  Check MCPO_API_KEY, Ruflo sandbox status, and proxy logs.
```

## Alert Destination Recommendation

Initial operator-beta cadence:

```text
Run manually after each deploy and once daily during active work.
Paste failures into Paperclip as a new incident issue.
```

Recommended next alert destination:

```text
Email or Slack webhook from a cron wrapper once v1.7D security review approves secret handling.
```

Do not put alert webhooks directly in the repository.
