# V2.0B Chat-to-PR Proof

V2.0B is the first GitHub write proof under the V2.0 production execution control plane.

The goal is intentionally narrow:

```text
approved chat plan -> generated branch -> docs-only file update -> GitHub pull request -> audit ledger -> graph memory packet
```

This is not general repository automation. It does not give an agent raw GitHub access. It creates a typed, scoped path for one approved PR candidate.

## Default Dry Run

```bash
cd projects/reliability-policy-matrix
make chat-to-pr-proof
```

The default config is:

```text
configs/chat_to_pr_v2_0B.json
```

By default, `live_enabled` is `false`, so the script does not call GitHub.

## Live Gates

A live PR is blocked unless all gates pass:

```text
VELOCE_CHAT_TO_PR_LIVE=1
live_enabled=true
GITHUB_TOKEN is set
GITHUB_REPOSITORY is set
proof_file starts with an allowed docs/memory prefix
```

The token is used only as a bearer token and is not printed in generated outputs.

## Scoped Actions

The live path uses the GitHub API to:

```text
read the base branch ref
create a generated branch
write one docs-only proof file
open one pull request
```

The committed proof file target is:

```text
docs/v2.0B-chat-to-pr-live-proof.md
```

The committed allowed prefixes are:

```text
docs/
knowledge/graph-memory/
```

## Outputs

```text
artifacts/derived/chat_to_pr_v2_0B.json
artifacts/derived/chat_to_pr_v2_0B.md
artifacts/derived/chat_to_pr_audit_v2_0B.jsonl
artifacts/derived/chat_to_pr_memory_v2_0B.md
```

The memory markdown is intended for the Obsidian -> Graphify loop, so the proof becomes queryable through OpenWebUI graph memory after extraction.

## Rollback

If a live proof creates the wrong PR or branch, the rollback is:

```text
close the generated PR
delete the generated branch
rerun graph extraction so memory reflects the correction
```

After V2.0B is accepted, the next safest proof is long-running job heartbeat before canary deploy.
