# V2.1-V2.5 Production AI OS Completion

## Summary

V2.1-V2.5 turns the V2.0 production control-plane proof into the first durable execution layer for Veloce Research OS.

The production posture remains conservative: live mutations are still capability-gated, token-scoped, and enabled one pilot at a time. The new layer adds the missing common substrate: a typed MCPO execution API, durable runner state, leases, idempotency, cancellation, audit tails, memory events, and packet schemas.

## What Changed

- V2.1 adds OpenWebUI/MCPO endpoints for production execution status, job enqueue, job status, job cancel, job approve, and audit tail.
- V2.2 adds `production_job_runner.py`, a file-backed durable runner for typed packets in `artifacts/derived/v2_jobs`.
- V2.3 keeps low-risk live pilots bounded to Paperclip writeback, docs-only chat-to-PR, and graph-memory ingestion.
- V2.4 keeps canary and rollback as real adapter slots but still gated behind explicit approval and health evidence.
- V2.5 adds packet schemas, redaction checks, forbidden-key checks, trace IDs, and OpenWebUI-friendly status flows.

## Safety Rules

- No endpoint accepts raw shell, raw Docker, raw token text, arbitrary filesystem paths, or untyped commands.
- Approval only changes `queued_dry_run` to `approved_for_live_candidate`; approval does not execute a job.
- The runner writes audit and memory records for every transition.
- Job packets, audit events, memory markdown, Paperclip comments, and PR bodies must remain secret-free.
- Live production mutation stays off unless the specific capability has an explicit live gate and scoped environment token.

## Runner State Machine

```text
queued_dry_run
approved_for_live_candidate
lease_acquired
running
heartbeat
completed
blocked
cancelled
rollback_required
rollback_prepared
```

## OpenWebUI Prompt Templates

- What is running? Call `veloce_production_execution_status`.
- What is blocked? Call `veloce_production_execution_status`, then inspect jobs whose `runner_state` is `blocked` or `rollback_required`.
- What needs approval? Call `veloce_production_execution_status`, then approve only a specific job with `veloce_production_job_approve`.
- What changed? Call `veloce_production_audit_tail` and then query `veloce_knowledge_graph_query` for the latest production execution memory.
