# V1.9L Operating Graph Ingestion Layer

Date: 2026-05-20

## Context

V1.9K made graph queries useful from OpenWebUI by ranking architecture and runbook documents ahead of test nodes. The next product step is to make the graph represent the whole Veloce operating system, not only source files.

The graph should contain:

- Paperclip issues, tasks, teams, dispositions, comments, and evidence references.
- Hermes memory and agent skills.
- Ruflo planning and orchestration skills.
- OpenWebUI tool methods.
- MCPO endpoint paths.
- Obsidian human-readable memory.
- Graphify machine-readable graph output.
- GitHub and VPS verification context.

## Implementation

V1.9L adds a secret-free graph-memory export script:

```bash
python3 tools/knowledge/graph_memory_export.py \
  --repo-root . \
  --out knowledge/graph-memory
```

It writes curated markdown nodes for Obsidian and Graphify:

```text
knowledge/graph-memory/veloce-operating-graph.md
knowledge/graph-memory/openwebui-mcpo-tool-surface.md
knowledge/graph-memory/paperclip-work-ledger.md
knowledge/graph-memory/hermes-ruflo-skills.md
knowledge/graph-memory/obsidian-graphify-memory-loop.md
knowledge/graph-memory/manifest.json
```

These files are intentionally markdown-first. Obsidian can show them to humans, and Graphify can extract them into `graphify-out/graph.json` for chat/agent queries.

## Paperclip Issue And Task Export

Live Paperclip data should enter the graph through a redacted metadata export, not raw database scraping.

Supported input:

```bash
python3 tools/knowledge/graph_memory_export.py \
  --repo-root . \
  --out knowledge/graph-memory \
  --paperclip-jsonl /path/to/secret-free-paperclip-export.jsonl
```

Expected JSONL shape:

```json
{"id":"VEL-123","title":"Close graph memory loop","status":"done","team":"AI Ops","project":"Veloce"}
```

The exporter only keeps small metadata fields and redacts secret-like strings. Bodies, raw comments, credentials, tokens, and environment values should not be exported.

## Product Meaning

The intended memory flow is:

```text
Paperclip issue/task metadata -> graph-memory markdown -> Obsidian
Hermes/Ruflo/OpenWebUI/MCPO capability metadata -> graph-memory markdown -> Obsidian
Obsidian markdown + repo docs + code -> Graphify graph.json
OpenWebUI -> MCPO knowledge_graph_query -> graph-grounded answer
Hermes/Ruflo -> graph context for reasoning and planning
```

This makes Veloce feel closer to infinite memory without giving the model raw production access. The model queries an evidence graph; humans can inspect the same memory in Obsidian.

## Acceptance

- Graphify includes explicit operating graph nodes under `knowledge/graph-memory/`.
- OpenWebUI graph queries can find product-level memory files before low-level tests.
- Paperclip issue/task metadata can be added through a secret-free JSONL export.
- Memory writes remain disabled by default in MCPO.
- Raw graph files are still not published to the public showcase without a redaction pass.
