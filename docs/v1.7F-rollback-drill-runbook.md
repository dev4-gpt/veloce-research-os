# V1.7F Production Rollback Drill Runbook

Date: 2026-05-20

## Purpose

This runbook performs a non-destructive rollback drill for the Veloce stack.

It does not delete production data, does not restore over live volumes, and does not start Ruflo runtime execution.

Security note:

```text
Do not paste `docker compose config` output into Paperclip, GitHub, Obsidian, or chat.
Rendered compose output can include environment secrets.
The rollback drill script suppresses compose stdout and writes only a success marker.
```

## Files

```text
projects/reliability-policy-matrix/scripts/rollback_drill.py
projects/reliability-policy-matrix/configs/rollback_drill_v1_7F.json
projects/reliability-policy-matrix/Makefile
```

## What The Drill Checks

```text
current git commit can be recorded
compose files exist and are checksummed
latest backup directory exists and SHA256SUMS passes
latest restore-test directory contains expected files
docker compose config renders without applying changes
production restore is not performed
production data is not deleted
```

## Run

Run on the VPS:

```bash
cd /root/veloce-research-os
git pull --ff-only

cd /root/veloce-research-os/projects/reliability-policy-matrix
make rollback-drill
make stack-health
```

Expected report:

```text
artifacts/derived/rollback_drill_v1_7F.json
```

Expected top-level fields:

```text
"ok": true
"mode": "non_destructive"
"production_restore_performed": false
"production_data_deleted": false
```

## Proxy Log Proof

`make stack-health` should be run after the rollback drill.

Then:

```bash
docker logs --tail=100 aiagency-mcpo-openwebui-proxy \
  | grep -E 'stack_status|repo_status|ruflo_status|ruflo_execution_packet'
```

Expected:

```text
POST /stack_status HTTP/1.1" 200
POST /repo_status HTTP/1.1" 200
POST /ruflo_status HTTP/1.1" 200
POST /ruflo_execution_packet HTTP/1.1" 200
```

## Destructive Action Gate

Do not run a destructive production restore unless all conditions are true:

```text
1. Human operator explicitly approves the restore.
2. Current production volumes are snapshotted first.
3. Target service is stopped.
4. Backup archive checksum passes.
5. Rollback archive exists.
6. Post-restore stack-health command is ready.
```
