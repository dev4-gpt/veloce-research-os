## Risk Table

| Area | Risk | Status | Owner | Action |
|---|---|---|---|---|
| MCPO | Bearer token exposure would expose safe tools | Controlled | VPS operator | Rotate `MCPO_API_KEY` after demos or suspected exposure |
| Open WebUI | Importing broad tools could create unsafe write surface | Controlled | Operator | Keep tool imports allowlisted |
| Paperclip | Agent API disposition writes are unreliable/401 | Controlled | Paperclip control-plane owner | Keep manual disposition until auth is intentionally restored |
| Hermes | Token-heavy agent context can cause unsuitable exact-output tests | Controlled | Operator | Use Hermes for memory/behavior, not tiny exact replies |
| Ruflo | Runtime has broad daemon/swarm/hooks/autopilot surface | Controlled | Operator | Keep runtime disabled and sandbox mounted read-only |
| GitHub | Write access should remain outside chat tools | Controlled | Repo owner | Use human-approved Codex/Git commits only |
| Kimi WebBridge | Browser hands can operate logged-in sessions | Pending | Operator | Evaluate only in separate Chrome profile with low-risk UI task |
| Backups | Local backup exists, off-host cadence not implemented | Warning | VPS operator | Add scheduled/off-host backup after approval |
| Monitoring | Manual monitor exists, alert webhook not configured | Warning | VPS operator | Add cron/email or Slack after secret-handling approval |

## Forbidden Open WebUI Capabilities

Do not expose these through Open WebUI:

```text
arbitrary shell
Docker control
filesystem write
GitHub write
Paperclip mutation
secret read
Ruflo daemon start
Ruflo swarm start
Ruflo hooks execution
Ruflo autopilot
unrestricted URL fetch
```

## Secret Rotation

| Secret | Owner | Rotate When |
|---|---|---|
| `MCPO_API_KEY` | VPS operator | after demos, team changes, suspected exposure |
| `OPENWEBUI_SECRET_KEY` | VPS operator | if Open WebUI sessions/auth are compromised |
| `API_SERVER_KEY` | VPS operator | if Hermes endpoint key is exposed |
| `NVIDIA_API_KEY` | VPS operator/provider owner | if usage anomalies or exposure occur |
| Paperclip API key | Paperclip control-plane owner | before enabling agent issue mutation |
| Future alert webhook | VPS operator | before/after incident or team changes |

Do not paste secret values into Paperclip, GitHub, Obsidian, OpenWebUI, or chat logs.

## Access Restrictions

Recommended:

```text
Keep MCPO behind bearer auth.
Keep OpenWebUI signup disabled.
Keep Ruflo sandbox read-only in MCPO proxy.
Keep Paperclip issue mutation manual until API auth is reviewed.
Keep GitHub writes outside OpenWebUI.
Keep Kimi WebBridge out of the production browser profile.
```
