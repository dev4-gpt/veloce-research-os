# V1.7 Production Readiness Plan

Date: 2026-05-20

## Current Answer

Veloce Research OS is working end to end for controlled operator use.

It is not yet production ready for unattended or customer-facing operation.

Current safe status:

```text
Operator beta: yes
Production-safe Ruflo bridge: yes
Autonomous production orchestration: no
Customer/SLA production readiness: no
```

## What Is Already Working

Verified path:

```text
Open WebUI
-> MCPO
-> Ruflo status, plan, execution packet
-> Paperclip handoff
-> Codex/GitHub/VPS verification
```

Verified services and bridges:

```text
Paperclip task ledger
Open WebUI operator cockpit
Hermes HTTP wrapper path
MCPO time tool
MCPO stack_status
MCPO repo_status
MCPO ruflo_status
MCPO ruflo_plan
MCPO ruflo_execution_packet
GitHub source of truth
VPS deployment proof
Obsidian knowledge mirror
```

Important safety decision:

```text
Ruflo does not run production work.
Ruflo produces status, plans, and approval-gated execution packets.
Actual execution remains human-approved through Codex/GitHub/VPS.
```

## Production Readiness Gap

The stack still needs:

```text
monitoring
alerting
backup and restore drills
secret rotation
access review
disaster recovery
release checklist
rollback checklist
Paperclip issue hygiene automation
dependency update policy
security review
load and failure testing
```

## V1.7 Target

V1.7 should make the system safe to operate repeatedly without relying on memory or ad hoc terminal history.

Target state:

```text
Every critical service has a health check.
Every data store has a backup and restore command.
Every deployment has a rollback path.
Every tool has proof logs.
Every Paperclip run has a disposition.
Every secret has a rotation note.
Every production-facing change has a GitHub commit and VPS verification.
```

## Production Gate

The project can be called production ready only after:

```text
1. v1.7A audit is done.
2. v1.7B backup and restore runbook is done.
3. v1.7C monitoring is running or documented with a scheduled operator cadence.
4. v1.7D security/access review is done.
5. One rollback drill is completed.
6. One Paperclip issue is fully closed with evidence from OpenWebUI, VPS logs, GitHub commit, and Obsidian mirror.
```

## Current Recommendation

Use the stack now for controlled operator workflows.

Do not use it yet for:

```text
unattended autonomous production changes
customer-facing SLAs
secret-bearing Ruflo workflows
automatic GitHub writes from chat
automatic Paperclip mutation from Ruflo
```

## Next Action

Create and run V1.7A first.

V1.7A should decide whether v1.7B backup/restore or v1.7C monitoring is the highest-risk next implementation task.
