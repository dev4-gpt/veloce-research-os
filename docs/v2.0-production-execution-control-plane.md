# V2.0 Production Execution Control Plane

Date: 2026-05-21

## Context

V1.8 and V1.9 made Veloce safe for controlled operator workflows, graph-backed memory, Hermes/Ruflo planning, OpenWebUI tool calls, stack health checks, and rollback drills.

V2.0 begins the execution layer:

- unattended production execution
- live Paperclip writeback
- chat-to-PR
- chat-to-canary deploy
- autonomous rollback execution
- real long-running agent job orchestration

The implementation intentionally starts with a control plane that fails closed. It creates durable job packets and audit records, but live execution is gated per capability.

## New Target

```text
operator_supervised_unattended_production
```

This means Veloce can prepare and queue unattended work safely, while each live production-writing capability must be enabled independently with scoped credentials and explicit environment gates.

## Control Plane

Run:

```bash
cd projects/reliability-policy-matrix
make production-execution
```

This executes:

```bash
python3 scripts/production_execution_control_plane.py \
  --config configs/production_execution_v2_0.json
```

Outputs:

```text
artifacts/derived/production_execution_v2_0.json
artifacts/derived/production_execution_v2_0.md
artifacts/derived/production_execution_audit_v2_0.jsonl
artifacts/derived/v2_jobs/*.json
```

## Capability Matrix

| Capability | Purpose | Default |
|---|---|---|
| `paperclip_writeback` | Post scoped comments and update dispositions | dry-run ready |
| `chat_to_pr` | Turn approved chat plans into GitHub PRs | dry-run ready |
| `chat_to_canary_deploy` | Promote approved candidates to canary | dry-run ready |
| `autonomous_rollback` | Restore last-known-good deployment and verify stack health | dry-run ready |
| `long_running_agent_jobs` | Queue resumable jobs with budgets, leases, and heartbeats | dry-run ready |
| `active_alerting` | Send alerts for failures, rollbacks, stale jobs, budgets, and spend anomalies | dry-run ready |

Live execution remains blocked unless:

```text
VELOCE_V2_LIVE=1
capability.live_enabled=true in config
required scoped env vars are present
kill switch is absent
risk policy allows the action
```

## Kill Switch

The kill switch path is:

```text
/root/ai-agency/AUTONOMY_DISABLED
```

If present, every capability decision becomes `blocked_kill_switch`.

## Live Gates

The config defines scoped live prerequisites:

```text
paperclip_writeback      -> PAPERCLIP_BASE_URL, PAPERCLIP_AUTOMATION_TOKEN
chat_to_pr               -> GITHUB_TOKEN, GITHUB_REPOSITORY
chat_to_canary_deploy    -> VELOCE_CANARY_APPROVED
autonomous_rollback      -> VELOCE_ROLLBACK_APPROVED
active_alerting          -> VELOCE_ALERT_WEBHOOK
long_running_agent_jobs  -> no secret required for dry-run job queue proof
```

The script records missing env var names only. It never prints secret values.

## Why This Solves The V2.0 Shape

This is the transition from planning-only autonomy to controlled execution:

```text
OpenWebUI chat intent
-> MCPO / Veloce Agentic Control
-> V2.0 capability decision
-> job packet
-> audit ledger
-> optional live capability
-> verification
-> rollback/alert/disposition
-> graph memory
```

V2.0 does not hand a model raw shell, raw Docker, or raw tokens. It makes production execution a set of typed, auditable, rollback-aware capabilities.

## Implemented Proof Order

1. V2.0A Paperclip scoped writeback against one demo issue.
2. V2.0B Chat-to-PR against a harmless docs-only branch.
3. V2.0C Long-running job heartbeat and stale-job alert.
4. V2.0D Canary deploy against a no-op candidate packet.
5. V2.0E Autonomous rollback proof against a last-known-good packet.
6. V2.0F Live Paperclip writeback pilot packet.
7. V2.0G Chat-to-PR live pilot packet.
8. V2.0H Canary deploy live pilot packet.
9. V2.0I Long-running agent runner proof.

Each proof should update the audit ledger and graph memory before expanding live scope.

## V2.0A Proof

The first proof is implemented in `docs/v2.0A-paperclip-scoped-writeback-proof.md`.

It narrows `paperclip_writeback` to:

```text
one issue comment
one issue disposition update
one JSONL audit record
one Obsidian/Graphify memory packet
```

The committed config stays dry-run safe. Live mode requires a local config copy with `live_enabled=true`, `VELOCE_PAPERCLIP_WRITEBACK_LIVE=1`, scoped Paperclip env vars, and verified Paperclip endpoint templates.

## V2.0B Proof

The second proof is implemented in `docs/v2.0B-chat-to-pr-proof.md`.

It narrows `chat_to_pr` to:

```text
one generated GitHub branch
one docs-only file update
one pull request
one JSONL audit record
one Obsidian/Graphify memory packet
```

The committed config stays dry-run safe. Live mode requires a local config copy with `live_enabled=true`, `VELOCE_CHAT_TO_PR_LIVE=1`, scoped GitHub env vars, and a proof file path under an allowed docs/memory prefix.

## V2.0C Proof

The third proof is implemented in `docs/v2.0C-long-running-job-heartbeat-proof.md`.

It narrows `long_running_agent_jobs` to:

```text
one resumable job packet
one heartbeat ledger record
one stale-job detector pass
one JSONL audit record
one Obsidian/Graphify memory packet
```

The committed config stays dry-run safe. Live-ready mode requires `live_enabled=true` and `VELOCE_LONG_JOB_LIVE=1`; no external token is required because V2.0C writes only the local job ledger.

## V2.0D Proof

The fourth proof is implemented in `docs/v2.0D-chat-to-canary-deploy-proof.md`.

It narrows `chat_to_canary_deploy` to:

```text
one no-op canary candidate packet
one pre-health snapshot
one post-health snapshot
one rollback packet
one alert packet
one JSONL audit record
one Obsidian/Graphify memory packet
```

The committed config stays dry-run safe. Live-ready mode requires `live_enabled=true`, `VELOCE_CANARY_DEPLOY_LIVE=1`, and `VELOCE_CANARY_APPROVED`. V2.0D still performs no production mutation; it proves the canary control packet before real deployment is allowed.

## V2.0E-I Pilot Pack

The V2.0E-I pack is implemented in `docs/v2.0E-I-production-pilot-pack.md`.

It adds:

```text
V2.0E autonomous rollback proof
V2.0F live Paperclip writeback pilot packet
V2.0G chat-to-PR live pilot packet
V2.0H canary deploy live pilot packet
V2.0I long-running agent runner proof
```

The committed configs remain dry-run safe. Live-capable paths require local config copies with `live_enabled=true`, explicit env gates, scoped credentials, and operator approval. V2.0E and V2.0H still do not run destructive production commands from the proof scripts; they prepare typed rollback/canary packets and evidence for review.
