# V1.9A-C OpenWebUI, Hermes, and Ruflo Closeout

Date: 2026-05-20

## Summary

V1.9A through V1.9C implement the first full-capability chat surface:

```text
OpenWebUI
-> MCPO proxy
-> Hermes memory/agent task bridge
-> Ruflo orchestration dry-run
-> Veloce capability boundary
```

This moves Veloce beyond isolated status tools and toward a single operator cockpit for governed agent work.

## Delivered

### V1.9A - OpenWebUI Full Tool Surface

Added:

```text
tools/openwebui/veloce_agentic_tool.py
```

OpenWebUI methods:

```text
veloce_stack_status
veloce_repo_status
veloce_ruflo_status
veloce_hermes_memory_query
veloce_hermes_agent_task
veloce_ruflo_orchestration_dry_run
veloce_ruflo_execution_packet
```

### V1.9B - Hermes Memory Tool Bridge

Added MCPO proxy endpoints:

```text
/hermes_memory_query
/hermes_agent_task
```

The bridge supports live Hermes calls when `API_SERVER_KEY` is available and dry-run validation when `dry_run=true`.

The bridge returns structured, secret-free responses with audit metadata.

### V1.9C - Ruflo Orchestration Dry Run

Added MCPO proxy endpoint:

```text
/ruflo_orchestration_dry_run
```

It returns:

```text
task_graph
worker_packets
denied_capabilities
capability_broker_summary
```

Ruflo runtime is not invoked. Production execution remains disabled.
