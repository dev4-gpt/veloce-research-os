# V3.2-V3.7 Live Production Rollout

Date: 2026-05-27

## Context

V2.6-V3.1 made the production AI OS pilot pack auditable and fail-closed. V3.2 begins the live rollout one capability at a time.

The rollout order is:

1. V3.2 Paperclip live writeback
2. V3.3 GitHub chat-to-PR live pilot
3. V3.4 Alert webhook live pilot
4. V3.5 Graph memory live ingestion
5. V3.6 Protected canary deploy
6. V3.7 Autonomous rollback execution gate

## V3.2 Paperclip Live Writeback

Run the wrapper in dry-run mode:

```bash
cd projects/reliability-policy-matrix
make paperclip-writeback-v3-2
```

The dry-run writes the local live wrapper config:

```text
projects/reliability-policy-matrix/artifacts/derived/paperclip_writeback_v3_2_live.local.json
```

When the operator is ready to run the live Paperclip pilot, use that local config:

```bash
cd projects/reliability-policy-matrix
VELOCE_PRODUCTION_AI_OS_LIVE=1 \
VELOCE_PAPERCLIP_WRITEBACK_LIVE=1 \
python3 scripts/paperclip_live_writeback_v3_2.py \
  --config artifacts/derived/paperclip_writeback_v3_2_live.local.json
```

`PAPERCLIP_BASE_URL` and `PAPERCLIP_AUTOMATION_TOKEN` must already be loaded in the shell. Do not paste or print token values.

The committed config remains dry-run safe. The live target is locked to:

```text
VEL-v2.0F-PILOT
```

The wrapper adds:

- exact target issue preflight
- global and stage live gates
- secret-free env presence checks
- previous disposition lookup when the Paperclip API exposes it
- V3.2 idempotency marker in the comment body
- trace id in report, audit, and memory
- rollback packet for partial failure
- local untracked wrapper config under `artifacts/derived/`
- local untracked effective Paperclip proof config under `artifacts/derived/`

Required live gates:

```text
VELOCE_PRODUCTION_AI_OS_LIVE=1
VELOCE_PAPERCLIP_WRITEBACK_LIVE=1
PAPERCLIP_BASE_URL
PAPERCLIP_AUTOMATION_TOKEN
live_enabled=true in a local untracked config copy
```

The first live mutation is exactly one comment and one disposition update.

## Later Stages

V3.3 creates one docs-only GitHub PR and does not merge it.

V3.4 sends one secret-free alert webhook event.

V3.5 ingests completed live events into Obsidian/Graphify memory and verifies OpenWebUI graph query discovery.

V3.6 promotes only an allowlisted protected canary candidate after Paperclip, PR, alerting, and graph ingestion evidence exist.

V3.7 runs only an allowlisted rollback adapter with explicit rollback approval and a last-known-good reference.
