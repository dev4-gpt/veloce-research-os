# V1.8 Production-Level Next Steps

Date: 2026-05-20

## Current State

The stack is production-hardened for controlled operator workflows.

```text
OpenWebUI cockpit: ready
MCPO safe tool bridge: ready
Ruflo planning/packet bridge: ready
Paperclip issue/evidence workflow: ready with manual disposition
Backup/restore proof: ready
Monitoring proof: ready
Security/access review: ready
Rollback drill: ready
Traefik route repair: ready
Unattended autonomous production: not ready yet
```

The autonomy readiness gate now focuses only on the remaining unattended-production controls. Secret rotation remains a security operations requirement, but it is no longer counted as an autonomy-architecture gate.

## Priority Order

### 1. Paperclip Scoped Write Auth

Goal: stop manual disposition/recovery loops.

Deliver:

```text
Scoped token or supported automation identity.
Can post comments.
Can set status/disposition.
Cannot read secrets or mutate unrelated projects.
Writes are rate-limited and logged.
```

Why first:

```text
Without reliable Paperclip writes, unattended runs cannot close the loop or leave durable status.
```

### 2. Capability Broker

Goal: replace broad tools with typed, narrow actions.

Deliver:

```text
read_stack_status
read_repo_status
read_ruflo_status
create_execution_packet
post_paperclip_comment
set_paperclip_disposition
open_github_pr
run_verification_suite
send_alert
```

Forbidden:

```text
arbitrary shell
raw Docker control
raw filesystem writes
raw GitHub token access
secret reads
Ruflo daemon/swarm/hooks/autopilot
browser extension production writes
```

### 3. Policy Gate

Goal: make every action pass through a risk decision before execution.

Risk classes:

```text
read_only: allow
comment_only: allow after Paperclip scoped-token proof
low_risk_write: PR/canary only
production_write: canary + rollback + alert required
destructive: human approval required
secret_bearing: human approval required
```

### 4. Audit Ledger

Goal: every autonomous action leaves a secret-free record.

Fields:

```text
action_id
issue_id
actor
capability
policy_decision
input_hash
output_hash
commit_sha
verification_result
rollback_result
alert_result
disposition
```

### 5. Active Alerting

Goal: move from manual checks to scheduled production monitoring.

Deliver:

```text
scheduled make stack-health
failure alert
rollback alert
stale issue alert
loop/budget alert
provider spend alert
```

### 6. Canary and Rollback Promotion

Goal: allow one low-risk change to move through a guarded deployment path.

Deliver:

```text
pre-change snapshot
candidate deploy or dry-run render
post-change stack-health
automatic rollback command
rollback verification
operator alert
```

### 7. Kill Switch

Goal: one documented command disables autonomy.

Deliver:

```text
pause scheduled controller
revoke automation token
leave OpenWebUI/Paperclip/Hermes/MCPO core services running
verify make stack-health still passes
```

## Production-Level Definition

Call the system production-level for unattended autonomy only when:

```text
make autonomy-readiness returns ok=true
one unattended dry run completes
one low-risk PR/canary flow completes
rollback proof exists
alert proof exists
audit ledger proof exists
manual kill switch proof exists
```
