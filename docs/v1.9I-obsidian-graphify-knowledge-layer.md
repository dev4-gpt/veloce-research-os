# V1.9I Obsidian + Graphify Knowledge Layer

Date: 2026-05-20

## Current State

Obsidian is already present in Veloce as a human-readable mirror:

```text
Paperclip/GitHub evidence -> Obsidian notes
operator notes -> Obsidian vault
runbooks and closeouts -> human research memory
```

That is useful, but it is not yet a first-class data layer. Today, Obsidian is mostly a durable notebook and evidence mirror. The system does not yet query the Obsidian graph from OpenWebUI, Hermes, Ruflo, or the capability broker.

## Target State

Add Graphify as the machine-readable graph layer between the repo, Obsidian, and the agent cockpit:

```text
GitHub repo + docs + Paperclip exports + Obsidian vault
-> Graphify extraction
-> graphify-out/graph.json
-> GRAPH_REPORT.md
-> graph.html
-> Obsidian graph export
-> optional MCP graph server
-> OpenWebUI / Hermes / Ruflo graph queries
```

## Why This Matters

Veloce has several memory surfaces:

```text
GitHub = source of truth
Paperclip = work ledger
Obsidian = human knowledge mirror
Hermes = agent memory/reasoning layer
Ruflo = planner/orchestrator
OpenWebUI = operator cockpit
```

Without a graph layer, each surface has partial memory. Graphify can turn the combined corpus into a relationship graph:

```text
which docs mention Ruflo
which tasks depend on Paperclip scoped writes
which runbooks prove production readiness
which components connect OpenWebUI to Hermes
which blockers are repeated across v1.7, v1.8, and v1.9
```

This makes Veloce stand out as a product: the system can navigate its own operating history.

## Public-Safe Boundary

Do not index secrets, `.env` files, local browser profiles, raw Docker compose renders, VPS logs with tokens, or private Paperclip exports.

The repo root includes `.graphifyignore` for this reason.

## Recommended Local Commands

Install Graphify from the official package:

```bash
uv tool install graphifyy
```

Build the repo graph:

```bash
cd /root/veloce-research-os
graphify . --no-viz
```

Build an interactive graph:

```bash
graphify .
```

Generate an Obsidian-oriented graph export:

```bash
graphify . --obsidian
```

Generate a call-flow architecture page:

```bash
graphify export callflow-html
```

Query the graph:

```bash
graphify query "what connects OpenWebUI, Hermes, Ruflo, Paperclip, and MCPO?"
graphify query "which documents prove v1.9A-C is deployed?"
graphify path "OpenWebUI" "Ruflo"
graphify explain "capability broker"
```

Optional MCP server:

```bash
python -m graphify.serve graphify-out/graph.json
```
