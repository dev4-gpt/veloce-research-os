The system is a control loop, not a chatbot.

The model never needs raw production access. It calls named, auditable capabilities through MCPO, and every serious action can be planned, checked, verified, and recorded.

Cockpit

OpenWebUI

The operator-facing interface where humans ask for stack checks, memory queries, orchestration plans, and execution packets.

Gateway

MCPO Proxy

A safe HTTP tool gateway that exposes only selected endpoints for OpenWebUI and keeps raw production access out of the model path.

Memory

Hermes

The reasoning and memory layer for project state, research context, and agent behavior experiments.

Planner

Ruflo

A governed orchestration planner that returns task graphs and worker packets while runtime execution stays disabled until gates pass.

Ledger

Paperclip

The task and evidence ledger for issues, dispositions, comments, closeouts, audit notes, and operator handoffs.

Runtime

VPS + GitHub

The VPS proves service health; GitHub remains source of truth for code, docs, runbooks, tests, and deployment manifests.

Graph

Obsidian + Graphify

Obsidian mirrors human-readable evidence while Graphify turns the repo, docs, and runbooks into a queryable graph, verified on VPS with 302 nodes and 403 edges.

What unlocks the full product.

Veloce is already strong for controlled operator workflows. The next stage is scoped live-loop automation: Paperclip writes, chat-to-PR, canary promotion, and audited VPS capabilities.

v1.9D

Scoped Paperclip writes

Allow the system to post issue evidence and update disposition with least-privilege credentials and audit trails.

v1.9E-F

VPS runner + chat-to-PR

Move from dry-run packets to controlled PR creation, verification, and operator-approved canary actions.

v1.9G-H

Demo loop

Close one issue end to end with chat, tools, evidence, rollback proof, Paperclip closeout, and investor demo script.

v1.9I

Knowledge graph layer

Mirror repo and evidence into Obsidian, then use Graphify to generate graph reports, call-flow maps, and queryable architecture context.

v1.9J

Graph memory loop

OpenWebUI can query graph status, ask graph questions, and generate secret-free memory packets for Obsidian and Graphify ingestion.

v1.9K

Docs-first graph answers

Graph queries now prioritize architecture, runbooks, and product memory for chat users while preserving raw graph matches for deeper inspection.

v1.9L

v1.9M

v1.9N
